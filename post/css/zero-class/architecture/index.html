Introduction

OOCSS, BEM, and Atomic CSS all have the same problem, they all utilize the class selector, and none have taken the time to solve it. Here lies the problem, we need a new architecture free for the class selector. I name this zero-class Architecture.

Architecture styles

lets have a quick overview of the current architectural styles before I explain why the class selector may not be the best option for styling.

OOCSS (Object-Oriented CSS)

OOCSS gets its cues from object-oriented programming. It does so by separating structure and skin. The purpose of OOCSS is to create stylesheets that are flexible, modular, and interchangeable. 

.box {
 padding: 5px;
}
.box-header {
 font-size: 16px;
 color: red;
}
.box-body {
 font-size: 12px;
 color: #fff;
 background-color: red;
}

<div class="box">
 <div class="box-header"></div>
 <div class="box-body"></div>
</div>

BEM (Block, Element, Modifier)

In BEM selectors are organized by block, a top-level abstraction of a component. Element, a component inside the block that performs a function and modifier, a variation of the block component. 

.block {
 padding: 5px;
}
.block__element {
 font-size: 16px;
 color: red;
}
.block--modifier {
 font-size: 12px;
 color: #fff;
 background-color:  red;
} 

<div class="block block__element block--modifier">
</div>

Atomic CSS

Atomic CSS gave birth to utility-first frameworks such as Tailwind. They are filled with single-purpose classes such as flex, pt-4, text-center, and rotate-90, which is no different from writing inline styles. This type of architecture ends by polluting your HTML file with arbitrary classes that make it hard to read and follow. zero-class Architecture can fix this unorganized string of classes more on that later. 

<div class="flex flex-column flex-wrap flex-gap flex-grow-children">
</div>

Atomic CSS cannot scale by itself to mitigate this problem Tailwind uses PostCSS plugins like autoprefixer and PurgeCSS. Additionally, it provides the @apply, which defeats the purpose of Atomic CSS.

SMACSS and ITCSS

This type of architecture does not define how to write CSS selectors rather they help to organize CSS rules in categories. They are both compatible with OOCSS, BEM, and Atomic CSS however they do not fix the problem of styling components with class names.

zero-class Architecture

I propose a new architecture that does not require class names for styling. The class selector by itself cannot be styled or extended with custom names. Data attributes have an advantage over the class selector because the * in data-* can be replaced by any name. We can use custom data attributes to categorize our components give them base styles, and create variations with different values. With data attributes, the same OCSS example becomes: 

[data-box] {
 padding: 5px;
}
[data-box="header"] {
 font-size: 16px;
 color: red;
}
[data-box="body"] {
 font-size: 12px;
 color: #fff;
 background-color: red;
}
                    
<div data-box>
 <div data-box="header"></div>
 <div data-box="body"></div>
</div>

Notice that we can use the data-box attribute by itself without a value. The BEM example: 

[data-block] {
 padding: 5px;
}
[data-block~="element"] {
 font-size: 16px;
 color: red;
}
[data-block~="modifier"] {
 font-size: 12px;
 color: #fff;
 background-color: red;
}

<div data-block="element modifier">
</div>

Utility classes can be grouped as data attributes, eliminating the need to prefix selectors such as flex-*, flex-column, flex-wrap, flex-gap, etc. 

[data-flex] {
 display: flex;	   
}
[data-flex~="column"] {
 flex-direction: column;
}
[data-flex~="wrap"] {	  
 flex-wrap: wrap;	  
}
[data-flex~="grow-children"] > * {	  
 flex-grow: 1;	  
}
[data-flex~="gap"] {	   
 gap: 10px;
}

<div data-flex="column wrap grow-children gap">
</div>

Much more simple and aesthetically pleasing. It is also possible to use JavaScript for styling certain utilities such as margins and paddings, completely removing them from the stylesheet. 

<div data-margin="5px 0" data-padding="5px 10px 5px 8px">
</div>

There are some rules to follow before naming a custom data attribute.

  • The name must not start with xml (case-insensitive).
  • The name must not contain any colon characters (:).
  • The name must not contain any capital letters.

Conclusion

When Marianne Brandt started her studies at the Bauhaus in 1923, she decided to burn all her previous works recognizing how traditional they had been. Styling with the class selector is our tradition. We must move away from styling with classes and start thinking about other ways of doing so. We can group similar selectors with data attributes, which is easier to read. Nowhere does the spec prohibits the use of data attributes for styling.

Lens camera manufacturers do not specify that a 50mm or an 85mm lens should be used only for portrait photography. All lenses can shoot portrait or landscape photography regardless of the focal length. These are only conventions people have and are no strict rules to follow. I can use an 85mm lens for landscapes or a fish eye lens for portraits as much as I can use data attributes for styling.

If you haven't noticed yet, this website uses data attributes for styling, if you are interested in this methodology, I have started a small framework to get you started. Visit RAMs, a zero-class framework.
